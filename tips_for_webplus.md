## 站群迷惑行为

（官方文档对于各种问题和站群的操作说了和没说没啥区别，而且有些地方说的不对）

站群 bug 贼多，功能难用，默认样式丑。**涉及一些保密信息，无法提供必要的截图**

1. 模板上传

    - 上传的图片不能含有中文，html 文件请改为 htm
    - list column 页的 css 一定要命名为 listColumn.css

2. 备份
    - 没有任何用处，备份的是你上次的 htm 等等文件夹，不会对你的拖拽的东西进行备份

3. 可视化拖拽页面
    - 火狐无法拖拽，请使用 chrome，ie 会非常卡不要用
    - 有时候拖着拖着就不能拖了，请刷新页面重新来
    - 网页时而会崩溃，这是常态
    - 拖进去的时候会有默认的样式显示，选择左边的栏目是将内容和信息绑定
    - 可视化拖拽页面有时会和略览页面显示不同（对于某些东西显示不出来），请以略览为准
    - 当拖入空的组件时会导致可以拖拽的窗口不可见，请 f12 修改一下窗口大小，有时修改 z-index
    - 多人修改同一页面可能会出现问题

4. frag  的书写

    - 必须为窗口 xx(xx 为数字)，当然还可以是面板 xx 不过没啥用，如 `窗口11`
    - 窗口 xx 不允许存在嵌套

5. 图片上传

    - 上传图片的时候一定要指定为原样式（原大小），否则会进行压缩，并生成以 **`_s`** 为结尾的副本。但上传文章标题的时候不用进行修改。

6. 部署

    - 不要好奇为啥你的修改没有，站群有很长时间的缓存并且无法强制刷新，等等就好了。

## 灵活运用组件
1. 带图片/文字的超链接
    - 单发一篇文章，文章里上传一张图片/写一段话然后点源码在外面嵌一层 a 标签即可，用新闻内容这个组件拖。这个方法也可以用于对文章内容居中（在源码最外层嵌入`<div>`，加上`style=text-align:center`）


2. 单显示一张图片
    - 法1：发一篇只有一个图片的文章，用新闻内容这个组件拖
    - 法2：修改图文标题的样式，在拖进去的高级配置里修改

3. 导航自定义样式
    - 在模板 -> 样式菜单 -> 导航管理里新增样式即可。（注：拖导航菜单的时候也可以加，但加了就不能改了）

4. 链接
    - 站群不允许使用你在任何html中使用 a 标签进行链接，只能使用站群给定的方法，在发文章的时候选择这是一个链接，链接外部网站就输入对应的网址，内部链接在最右边的一个小按钮上，选择栏目即可。
